# Development Resources (Canonical)

Use these references by default across all in-house development work.

## UI Component System
- shadcn/ui: https://ui.shadcn.com/

## Usage Policy
- Prefer shadcn/ui primitives for app UI scaffolding and component consistency.
- Keep styling practical and accessible; avoid over-customizing until core workflows are stable.
- When creating new frontend features, check shadcn/ui first before building custom components from scratch.
